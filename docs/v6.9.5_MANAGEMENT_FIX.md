## 💡 **API 设计理解**

### **accountUpdate 的双重功能**

从错误消息 "Field newStatus required for status change" 可以看出：

**accountUpdate API 同时处理两件事**：
1. **更新费率** - 通过 `<plan>` 对象
2. **更新状态** - 通过 `<newStatus>` 字段

**这意味着**：
- 不能只更新费率而不指定状态
- `newStatus` 是必需字段
- 通常设置为 `ACTIVE`（启用状态）
