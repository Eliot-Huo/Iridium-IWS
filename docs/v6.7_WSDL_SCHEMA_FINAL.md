## 🔧 v6.7 修正內容（2 項）

### **1. getSBDBundles 加入 serviceType** ⭐⭐⭐

**問題**：伺服器回應 "No plan provided" 錯誤

**修正前（v6.6 - 導致錯誤）**：
```xml
<tns:getSBDBundles>
    <request>
        <iwsUsername>IWSN3D</iwsUsername>
        <signature>...</signature>
        <serviceProviderAccountNumber>200883</serviceProviderAccountNumber>
        <timestamp>...</timestamp>
        <!-- 缺少 serviceType -->
    </request>
</tns:getSBDBundles>
```

**修正後（v6.7 - 成功）**：
```xml
<tns:getSBDBundles>
    <request>
        <iwsUsername>IWSN3D</iwsUsername>
        <signature>...</signature>
        <serviceProviderAccountNumber>200883</serviceProviderAccountNumber>
        <timestamp>...</timestamp>
        <serviceType>SHORT_BURST_DATA</serviceType>  <!-- ✓ 新增 -->
    </request>
</tns:getSBDBundles>
```

---

### **2. updateSubscriberSbdPlan 修正 plan 標籤順序** ⭐⭐⭐

**問題**：WSDL Schema 要求嚴格的標籤順序

**修正前（v6.6 - 順序錯誤）**：
```xml
<plan>
    <sbdBundleId>17</sbdBundleId>
    <demoAndTrial>0</demoAndTrial>
    <!-- 缺少 lritFlagstate 和 ringAlertsFlag -->
</plan>
```

**修正後（v6.7 - Schema 合規）**：
```xml
<plan>
    <lritFlagstate>0</lritFlagstate>       <!-- 1. 必填 (Long) -->
    <demoAndTrial>0</demoAndTrial>         <!-- 2. 必填 (Long) -->
    <sbdBundleId>17</sbdBundleId>          <!-- 3. 必填 (Long) -->
    <ringAlertsFlag>0</ringAlertsFlag>     <!-- 4. 必填 (Long) -->
</plan>
```

**Schema 順序要求**（嚴格）：
1. `lritFlagstate` - LRIT Flag State（Long 型別）
2. `demoAndTrial` - Demo and Trial（Long 型別）
3. `sbdBundleId` - SBD Bundle ID（Long 型別）
4. `ringAlertsFlag` - Ring Alerts Flag（Long 型別）

**所有值都使用數字**：
- `0` = False
- `1` = True
