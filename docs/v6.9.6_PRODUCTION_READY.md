## 💡 **PENDING 状态的关键理解**

### **1. PENDING 是衍生状态（Derived State）**

```
✅ 系统自动计算
✅ 无法手动设置
✅ 表示后台有未完成的开通订单
```

**从文档中**：
- "PENDING is a derived state"
- "Cannot be manually set via API"
- "Indicates outstanding provisioning service orders"

### **2. PENDING 状态下的限制**

```
❌ 无法修改账号（No updates allowed）
❌ 无法执行 accountUpdate
❌ 无法变更状态
❌ 设备无法使用（Cannot generate traffic）
```

**从文档中**：
- "No updates are allowed while the account status is PENDING"
- "The device cannot be modified or used"

### **3. 解除 PENDING 的方式**

```
✅ 自动转化：系统完成后台流程后自动变为 ACTIVE
⏳ 等待与重试：建议稍后重试（Retry later）
📞 联系支持：长时间卡住需要联系 Tier 2 Support
```

**从文档中**：
- "Status automatically changes to ACTIVE once provisioning completes"
- "Please retry the request later"
- "Contact Iridium Tier 2 Support if stuck"

```python
def update_subscriber_plan_with_retry(self, 
                                     imei: str,
                                     new_plan_id: str,
                                     max_retries: int = 3,
                                     retry_delay: int = 30,
                                     check_pending: bool = True):
    """
    变更费率（带 PENDING 检查和重试）
    
    Args:
        check_pending: 是否检查 PENDING 状态
        max_retries: 最大重试次数
        retry_delay: 重试间隔（秒）
    """
    
    if check_pending:
        # 检查是否在 PENDING
        status = self.check_account_status(imei)
        
        if status['status'] == 'PENDING':
            # 等待并重试
            for i in range(max_retries):
                print(f"Account is PENDING, retry {i+1}/{max_retries}...")
                time.sleep(retry_delay)
                
                status = self.check_account_status(imei)
                if status['status'] != 'PENDING':
                    break
            
            if status['status'] == 'PENDING':
                raise IWSException(
                    "Account is still in PENDING status after retries. "
                    "Please wait and try again later."
                )
    
    # 执行更新
    return self.update_subscriber_plan(imei, new_plan_id)
```
